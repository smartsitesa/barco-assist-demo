# Barco Assist Demo Spec

Goal: create a dedicated Barco Assist demo from the PDF website breakdown using the same premium design language as the Barco Group test site.

## Document-driven scope
- Domain target: barcoassist.co.za
- Build pages/sections for:
  - Landing welcome page
  - Maintain page
  - Hygiene page
  - Technology page
  - Conisoiur page
  - Contact page with AI assist chat positioning
- Keep structure ready for later software integration messaging.

## Core content from the PDF
- Landing page should combine About Us, B2B, B2C, affiliates / clients, and team in one welcome flow.
- Maintain page must include maintenance services and COC service support.
- Hygiene page presents hygiene-related service offering.
- Technology page presents tracking / visibility / reporting support.
- Conisoiur page remains cautious because final write-up is still missing.
- Contact page should support assist / emergency / after-hours routing once details are confirmed.
- Software section should frame future AI-enabled operational tooling without fabricating implementation claims.

## Design direction
- Match the Barco Group premium visual language:
  - dark architectural hero
  - gold + electric blue accents
  - glass cards and blueprint details
  - polished operational / facilities tone
- Make Barco Assist feel service-operations led rather than construction-led.

## Constraints
- Do not invent exact emergency numbers or staff contact details.
- Keep claims grounded in the provided PDF and visible Barco context.
- Stay static and deployable on GitHub Pages.

## Acceptance
- Clear service separation by section.
- Contact and AI assistant flows feel useful even with placeholder routing.
- Mobile layout works.
- Repo ready for push/deploy to SmartSite test GitHub account.
